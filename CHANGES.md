# Изменения: безопасность + новый функционал

## Ссылки, крафт, спойл
- **Вкладка «Ссылки»** — офицеры добавляют полезные ссылки с описанием и категорией, читают все. Кнопка «Загрузить набор» сидит дефолтные (вики Lu4, L2Hub, калькулятор-бот). Открытие в системном браузере (Rust-команда `open_url`). Коллекция `links` в правилах.
- **Вкладка «Крафт»** — калькулятор крафта сосок по данным из «Луч КП Дикей»: рецепты (Soulshot/Spiritshot D), редактируемые цены D-Crystal/Soul Ore/комиссии (сохраняются локально), расчёт себестоимости за штуку, профита за крафт, профита % и итога за N крафтов.
- **Роадмап → секция «Крафт и спойл»**: экономика сосок, рецепты, спойл-ресурсы (D-Crystal, Soul Ore, ресурсы на сет/пушку 34-35), математика нюков (когда соски в плюс: 2-3 нюка на моба — профит, 4+ — минус), кнопка на калькулятор. Крафтит Warsmith, спойлит Bounty Hunter (10-е позиции).

## Права роадмапа
- **Мемберы видят только текущую фазу** (по уровню пачки); ПЛ/офицеры — весь план.
- **Отмечать задачи выполненными могут только ПЛ и офицеры** — и в роадмапе, и на дашборде (чекбоксы у мемберов задизейблены). Правило БД `roadmapProgress` ужесточено до `isOfficer`.
- Советы ПЛ остаются видимыми всем (по решению).

## Авто-фаза по уровню пачки
- Активная фаза роадмапа определяется автоматически по **медианному уровню пачки** (уровни берутся из профилей → ростера; медиана устойчива к одному «забывчивому»). Никакого ручного ввода — мембер обновляет lvl в профиле при апе, остальное считает приложение.
- В «Роадмапе» активная фаза **подсвечивается** (бейдж «текущая» + рамка) и автоматически разворачивается при заходе.
- На **дашборде** — карточка «Сейчас по роадмапу: Фаза X (ур. N)» с целью фазы и невыполненными задачами (можно отмечать прямо с дашборда) + ссылка на полный роадмап.

## Встроенный детальный роадмап Lu4 (в приложении)
- Весь план из `ROADMAP_LU4.md` вынесен в структурированный модуль `src/data/lu4Roadmap.js` и рендерится во вкладке «Роадмап» — больше не markdown-файл, а живая система.
- Каждая фаза (0-6) — сворачиваемая, с секциями: 🎯 цель, ⏰ прайм / 🌙 оффпрайм, 🗺 зоны фарма, 📜 квесты (NPC/уровень/награда), 🛡 экипировка, 🐉 RB, ✅ задачи с **подзадачами** и советами, 💡 итоговые советы.
- **Общий прогресс на пачку** (`roadmapProgress`): любой не-гость отмечает задачи/подзадачи галочками, прогресс-бары по фазам и общий.
- **«Выдать в дашборд»** — офицер кнопкой раздаёт задачу роадмапа всей пати или конкретному игроку (в задачи дашборда, с тегом прайм/оффпрайм).
- Блок «О пачке и механиках»: кор-пати 9 слотов + 10-е позиции + ключевые механики Lu4.
- Состав пачки исправлен: **кор 9 = 3 Sorcerer, Shillien Elder, Elven Elder, Bishop, Overlord, Temple Knight, Blade Dancer** (Terramancer/Sword Singer → 10-е позиции).
- Правила БД: коллекция `roadmapProgress` (не-гость пишет только `done`).

## Фикс сворачивания в трей

## Настройки + полировка UI
- **Настройка крестика (✕)** — новый экран «Настройки» (шестерёнка в сайдбаре): выбор действия при закрытии окна — «спрашивать каждый раз» / «свернуть в трей» / «закрыть приложение». Хранится локально (`cp-close-action`), `DesktopIntegration` читает актуальное значение в момент закрытия.

## Редактор профиля мембера
- **Модалка профиля** (клик по аватару/нику в сайдбаре): аватар, игровой ник, уровень.
- **Аватар** — загрузка картинки, сжимается на клиенте в кружок 128×128 (data-URL JPEG). Без Firebase Storage и без правок CSP; хранится в `users/{uid}.avatar` и синхронизируется в слот ростера, поэтому виден всем (сайдбар, список «Мемберы»).
- **Уровень** — игрок сам ставит свой lvl; синхронизируется в слот ростера (ПЛ видит актуальные уровни в Ростере/Мемберах), в сайдбаре бейдж «Ур. N».
- Правила БД: игрок может менять `name/lvl/avatar` только в СВОЁМ слоте ростера (раньше синк ника в ростер у мемберов молча падал — тоже починено).
- Старый инлайн-редактор ника заменён полноценным профилем.

## Расписание прайм-тайма
- **Вкладка «Расписание»** — недельная сетка (Пн–Вс) с активностями по времени: тип (Фарм/РБ/Эпик/Осада/Олимп/Эвент/Другое), время, заметка. Офицеры создают/удаляют события.
- **«Кто идёт»** — каждый мембер отмечает участие кнопкой «Иду», у события виден счётчик и список ников. Посещаемость — документ на игрока (`scheduleAttendance/{userId}`), пишет только сам; события — `schedule` (офицеры).
- Правила БД: `schedule` (офицеры), `scheduleAttendance` (свой документ).

## Роадмап и старт сервера (для запуска lu4.org 31.07)
- **Вкладка «Роадмап»** — план развития пака по фазам (День 1 / Неделя 1 / Первый месяц…). Офицеры добавляют цели, любой мембер отмечает выполнение (общий прогресс), у каждой фазы прогресс-бар + общий прогресс сверху. Данные: коллекция `roadmap` (пункт = документ, отметка `done` — одно поле, без гонок).
- **Стартовый шаблон** — ПЛ одной кнопкой загружает готовый каркас плана и правит под свой сервер.
- **Обратный отсчёт до старта** — баннер на дашборде: «До старта lu4.org: N дн.», после запуска — «сервер запущен, день N». Дату ПЛ задаёт на странице роадмапа (`config/serverInfo.launchDate`).
- Правила БД: `roadmap` (офицеры управляют, не-гость может переключать только `done`).

## Безопасность (firestore.rules — переписаны)
- **Квесты мемберов исправлены.** Прогресс перенесён из общего `config/questLog` в подколлекцию `questProgress/{userId}`. Теперь каждый мембер пишет ТОЛЬКО свой документ (`questLogService.js` обновлён, формат для UI сохранён). Раньше запись мембера отклонялась правилами.
- **Валидация схемы во всех правилах:**
  - `transactions` — при создании проверяются `amount` (число ≥ 0), `type` ∈ {income, expense}, `currency` ∈ {adena, mc}, `member` (строка), `addedById == uid`. PL может править только `description/member/memberId` (суммы неизменяемы), удаление — только PL.
  - `roster` — `name/className` строки, `position` int, `lvl` число ≥ 1.
  - `tasks` — непустой `text`, `tag` ∈ {prime, offprime}.
- **Новые коллекции с правами:** `notes`, `news` (читают все не-гости, пишут офицеры+), `presence` (пишет только сам пользователь).

## Новый функционал
- **Заметки на дашборде** (`notes`) — офицеры/ПЛ пишут, видят все, сворачиваются (состояние в localStorage).
- **Вкладка «Новости»** (`/news`) — ПЛ/офицеры публикуют/редактируют/удаляют, читают все мемберы.
- **Онлайн-индикатор** — heartbeat раз в 45 сек (`presence`), точки онлайн/оффлайн в Ростере и Мемберах. Онлайн = активность за последние 2 минуты.
- **Казна** — в выборе мембера только реально назначенные игроки (без вакантных и заглушки «Занято»).
- **Мемберы** — при недоборе минимума адены показывается «нужно ещё X» (в строке и в раскрытии).
- **Трей** — меню (Открыть / Выход), клик по иконке разворачивает окно.
- **Крестик окна** — спрашивает «свернуть в трей или выйти» (`DesktopIntegration.jsx`).
- **Уведомления** — нативное уведомление о новой новости, когда окно не в фокусе.
- **Автообновление** — при запуске проверяет GitHub, скачивает и устанавливает новую версию, затем предлагает перезапуск (`restart_app`). Конфиг updater корректен (endpoint, pubkey, createUpdaterArtifacts, подписные ключи в CI).
