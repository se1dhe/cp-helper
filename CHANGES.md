# Изменения: безопасность + новый функционал

## Поиск: клик по рецепту открывает его в крафте
- Раньше клик по рецепту в глобальном поиске просто вёл на «Крафт» без выбора. Теперь id рецепта прокидывается через навигацию: «Крафт» открывается на вкладке рецептов с **уже выбранным предметом и раскрытой сметой** (базовые ресурсы + стоимость). Фильтры сбрасываются, чтобы предмет был виден в списке. Работает и по клику, и по Enter. Состояние навигации очищается, чтобы выбор не «прилипал» при возврате.

## Прогон на прочность перед стартом (закалка)
- **Оффлайн-устойчивость**: включён персистентный кэш Firestore (`persistentLocalCache`) с безопасным фолбэком в память — данные переживают перезапуск, приложение открывается и работает без сети. Синергия с single-instance (конфликта вкладок нет).
- **Гость больше не дёргает подписки** (новости/RB/presence) без прав чтения — гард `active` в `DesktopIntegration` убирает permission-denied в консоли и лишнюю работу.
- **Подстрахован `fallbackName`** при пустом email (индекс `split('@')[0]` не падает).
- Аудит подтвердил: вычисления на пустой базе не роняют UI (деления защищены `Math.max(1,…)`/`range||1`/ранние `return`, уровни/суммы через `Number()||…`, даты через `?.toDate?.()`, `packLevel`/`getActivePhaseId`/`getCountdown` возвращают безопасные дефолты). Крашей не найдено.

## Кастомная полоса заголовка окна (frameless)
- Убрана нативная рамка ОС (`decorations: false`), нарисован свой заголовок (`Titlebar.jsx`) — на macOS и Windows одинаково, под тему приложения.
- Слева логотип 0utLaw + «CP-Helper · UASQUAD»; по центру живые чипы: **счётчик до старта / «День N»** и **текущая фаза** пачки; справа свои кнопки **свернуть / развернуть / закрыть** (закрытие уважает настройку трей/спросить/выход).
- Вся полоса — зона перетаскивания окна (`data-tauri-drag-region`), даблклик — развернуть. Чипы видны только авторизованному не-гостю (правила Firestore).
- Права окна в `capabilities/desktop.json`: `minimize`, `toggle-maximize`, `is-maximized`, `start-dragging`. Rust-конфиг — вступает в силу после сборки.

## Аудит прав ролей (сверка фронта и firestore.rules)
- Полная проверка гейтов по ролям: бэкенд-правила Firestore — без дыр (записи ограничены офицером/ПЛ/владельцем, `roadmapProgress` пишут только офицеры, персональные документы — только владелец).
- **Поиск (Ctrl+K) стал ролезависимым**: страницы «Мемберы» (офицеры+) и «Ростер» (ПЛ) больше не показываются тем, кому недоступны; поиск по нику ведёт на «Мемберов» только офицерам — мембер больше не утыкается в «доступ запрещён».
- **Убрана мёртвая коллекция `roadmap`**: неиспользуемые функции сервиса и её блок правил удалены (актуален `roadmapProgress`) — меньше неоднозначности. Требует передеплоя правил (`release.sh` делает).

## Одна копия приложения (single-instance)
- Плагин `tauri-plugin-single-instance` (регистрируется первым): повторный запуск ярлыка не открывает вторую копию, а разворачивает и фокусирует уже запущенное окно (в т.ч. из трея). Rust-изменение — вступает в силу после сборки.

## Роадмап: пошаговость + привязка RB + экспорт
- **Пошаговый старт (1→19)** — отдельная секция со строго упорядоченным чек-листом (расовый квест → оружие → заряды → марш в Глудин → сбор → Will the Seal). Отмечают офицеры (общий прогресс).
- **Микро-таргеты по уровням** — в каждой фазе таймлайн «на уровне N сделай ровно это» (19 → Dragon Fangs, 20 → профа+обмен+Red-Eyed, 27 → Acts of Evil, 35 → Temple…).
- **Расовые маршруты 1-15** — по каждой расе (Люди/Эльфы/Т.Эльфы/Орки/Гномы) нумерованный порядок квестов с уровнями, наградами и **ссылками l2hub** (открываются в браузере).
- **RB по уровню пачки** — в «Боссах» актуальные по уровню боссы подсвечиваются и помечаются «по уровню» (медиана уровней пачки).
- **Экспорт роадмапа** — кнопка копирует роадмап с прогрессом (markdown) в буфер обмена, чтобы поделиться вне приложения.

## Батч 3 (UX/UI): поиск, drag-n-drop, тултипы, полировка
- **Глобальный поиск** — кнопка в сайдбаре + **Ctrl/Cmd+K**: палитра ищет по страницам, мемберам, рецептам (с иконками) и боссам; Enter — перейти к первому, Esc — закрыть.
- **Drag-n-drop ростера** — ПЛ перетаскивает карточки слотов, порядок (`position`) пересохраняется батчем; подсветка цели при перетаскивании.
- **Тултипы** — иконки классов показывают название класса, иконки предметов в калькуляторе — название предмета (по наведению).
- **Полировка** — единые кастомные скроллбары, фокус-обводка инпутов, плавные transitions на карточках/кнопках; светлая тема доведена по поверхностям (инпуты, hover, чипы).

## Батч 3 (часть): профиль игрока + дисциплина
- **Страница «Профиль»** (пункт меню): аватар, класс, уровень, роль; карточки — вклад в казну (адена, из транзакций), квесты done/total, стрик дисциплины; текущая фаза роадмапа по своему уровню; список своих квестов.
- **Стрик дисциплины** — в «Мемберы» добавлена колонка «Стрик» (🔥 N дней подряд с аденой ≥ минимума, считается из истории транзакций). Профиль тоже показывает стрик.

## Батч 2: графики казны, аудит-лог, тема
- **Графики казны** — сворачиваемый блок в «Казне»: линия баланса адены по дням (SVG) и топ-вклад по мемберам (бары). Без внешних библиотек.
- **Аудит-лог** — иммутабельная коллекция `audit`: логируются смена ролей, операции казны, действия по RB (добавил/убил). Просмотр в «Управлении» (офицеры). Правила: read офицеры, create только от своего имени, без правок/удаления.
- **Тёмная/светлая тема** — переключатель в «Настройках», применяется мгновенно и на старте (`data-theme`), сохраняется локально. Светлая тема — через override CSS-переменных.

## Батч 1: RB-таймеры, прайс-лист, заявки на крафт
- **Таймеры рейд-боссов** — вкладка «Боссы»: ПЛ/офицеры ведут (имя, уровень, респ в часах, локация), кнопка «Убит» стартует таймер, живой обратный отсчёт у всех. **Трей-уведомление за 15 минут** до респа и в момент спавна. Кнопка «Загрузить набор боссов 20-40». Коллекция `raidbosses` (офицеры пишут).
- **Общий прайс-лист сервера** — цены материалов теперь в `config/prices` (офицеры правят, все видят). Калькулятор рецептов тянет общий прайс; у мемберов поля read-only.
- **Заявки на крафт** — вкладка «Крафт → Заявки»: мембер ищет предмет, ставит кол-во/заметку и оставляет заявку; крафтер видит очередь, раскрывает **нужные базовые ресурсы** (движок рецептов) с суммой по общему прайсу; статус «сделано» и удаление у автора/офицеров. Коллекция `craftRequests`.

## Права роадмапа
- **Мемберы видят только текущую фазу** (по уровню пачки); ПЛ/офицеры — весь план.
- **Отмечать задачи выполненными могут только ПЛ и офицеры** — и в роадмапе, и на дашборде (чекбоксы у мемберов задизейблены). Правило БД `roadmapProgress` ужесточено до `isOfficer`.

## Авто-фаза по уровню пачки
- Активная фаза роадмапа определяется автоматически по **медианному уровню пачки** (уровни берутся из профилей → ростера; медиана устойчива к одному «забывчивому»). Никакого ручного ввода — мембер обновляет lvl в профиле при апе, остальное считает приложение.
- В «Роадмапе» активная фаза **подсвечивается** (бейдж «текущая» + рамка) и автоматически разворачивается при заходе.
- На **дашборде** — карточка «Сейчас по роадмапу: Фаза X (ур. N)» с целью фазы и невыполненными задачами (можно отмечать прямо с дашборда) + ссылка на полный роадмап.

## Редактор профиля мембера
- **Модалка профиля** (клик по аватару/нику в сайдбаре): аватар, игровой ник, уровень.
- **Аватар** — загрузка картинки, сжимается на клиенте в кружок 128×128 (data-URL JPEG). Без Firebase Storage и без правок CSP; хранится в `users/{uid}.avatar` и синхронизируется в слот ростера, поэтому виден всем (сайдбар, список «Мемберы»).
- **Уровень** — игрок сам ставит свой lvl; синхронизируется в слот ростера (ПЛ видит актуальные уровни в Ростере/Мемберах), в сайдбаре бейдж «Ур. N».
- Правила БД: игрок может менять `name/lvl/avatar` только в СВОЁМ слоте ростера (раньше синк ника в ростер у мемберов молча падал — тоже починено).
- Старый инлайн-редактор ника заменён полноценным профилем.

## Роадмап и старт сервера (для запуска lu4.org 31.07)
- **Вкладка «Роадмап»** — план развития пака по фазам (День 1 / Неделя 1 / Первый месяц…). Офицеры добавляют цели, любой мембер отмечает выполнение (общий прогресс), у каждой фазы прогресс-бар + общий прогресс сверху. Данные: коллекция `roadmap` (пункт = документ, отметка `done` — одно поле, без гонок).
- **Стартовый шаблон** — ПЛ одной кнопкой загружает готовый каркас плана и правит под свой сервер.
- **Обратный отсчёт до старта** — баннер на дашборде: «До старта lu4.org: N дн.», после запуска — «сервер запущен, день N». Дату ПЛ задаёт на странице роадмапа (`config/serverInfo.launchDate`).
- Правила БД: `roadmap` (офицеры управляют, не-гость может переключать только `done`).

## Доработки по фидбеку (входят в 1.3.0)
- **Двойная иконка в трее** — убран `trayIcon` из `tauri.conf.json` (он дублировал иконку без меню). Осталась одна, создаваемая в Rust, с рабочим ПКМ-меню.
- **Третья карточка дашборда** — «В сети»: счётчик онлайн-мемберов (X / всего) со списком ников. Сетка карточек и так была на 3 колонки, поэтому раньше пустовала.
- **Заметки → карточки** — заметки офицеров/ПЛ теперь отображаются плиткой карточек, а не строками.
- **Личные задачи** — ПЛ может назначить задачу конкретному игроку (селект «Всей пати / игрок»). Мембер видит общие задачи + свои личные; личные помечены бейджем с ником. Правила БД дают не-гостю переключать `done` только у общей или своей задачи.

## Безопасность (firestore.rules — переписаны)
- **Квесты мемберов исправлены.** Прогресс перенесён из общего `config/questLog` в подколлекцию `questProgress/{userId}`. Теперь каждый мембер пишет ТОЛЬКО свой документ (`questLogService.js` обновлён, формат для UI сохранён). Раньше запись мембера отклонялась правилами.
- **Валидация схемы во всех правилах:**
  - `transactions` — при создании проверяются `amount` (число ≥ 0), `type` ∈ {income, expense}, `currency` ∈ {adena, mc}, `member` (строка), `addedById == uid`. PL может править только `description/member/memberId` (суммы неизменяемы), удаление — только PL.
  - `roster` — `name/className` строки, `position` int, `lvl` число ≥ 1.
  - `tasks` — непустой `text`, `tag` ∈ {prime, offprime}.
- **Новые коллекции с правами:** `notes`, `news` (читают все не-гости, пишут офицеры+), `presence` (пишет только сам пользователь).

## Новый функционал
- **Заметки на дашборде** (`notes`) — офицеры/ПЛ пишут, видят все, сворачиваются (состояние в localStorage).
- **Вкладка «Новости»** (`/news`) — ПЛ/офицеры публикуют/редактируют/удаляют, читают все мемберы.
- **Онлайн-индикатор** — heartbeat раз в 45 сек (`presence`), точки онлайн/оффлайн в Ростере и Мемберах. Онлайн = активность за последние 2 минуты.
- **Казна** — в выборе мембера только реально назначенные игроки (без вакантных и заглушки «Занято»).
- **Мемберы** — при недоборе минимума адены показывается «нужно ещё X» (в строке и в раскрытии).
- **Трей** — меню (Открыть / Выход), клик по иконке разворачивает окно.
- **Крестик окна** — спрашивает «свернуть в трей или выйти» (`DesktopIntegration.jsx`).
- **Уведомления** — нативное уведомление о новой новости, когда окно не в фокусе.
- **Автообновление** — при запуске проверяет GitHub, скачивает и устанавливает новую версию, затем предлагает перезапуск (`restart_app`). Конфиг updater корректен (endpoint, pubkey, createUpdaterArtifacts, подписные ключи в CI).
